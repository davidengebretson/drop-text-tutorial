Here I will display some drop text (also known as an Accordion component). This implementation is much more simple in react, but it is nice to know how to do this type of component in pure html, css, and JavaScript. Enjoy and Happy coding.

In This tutorial I will demonstrate coding tips and techniques oriented towards blind developers. This will also show the world How blind developers can succeed in full stack development tasks.

The drop buttons (Accordions) I place within "drop panels" but, in reality, they can be placed anywhere. See the How To drop panel for implementation instructions. See detailed instructions drop panel for a list of how to develop them from scratch yourself.

Step 2b Layout HTML

Here is where you actually type out the HTL you will basically need. There will be alot of div's and you as a blind programmer will have to keep them straight.

So you will rely heavily on the keystroke fold (control + k, control + l). This toggles the code and you end up with 2 lines of folded code. Means cutting and moving on requires you to selct 2 lines and move anywhere and paste it.

I also make sure to to place identify attributes to the left. This way I hear what is imortant first when I go tdown the html. Example: div id="drop-r0" means to me this is a right drop panel and the 1st one at level 0."

I use emmitters to create id, class attributes and sometimes others. example: div#drop-r0.drop-panel followed the tab key will generate the id and class for me. Less chance for type-o's. I press enter so I can put content in between.

With that in mind We will need Outer div for the drop panel, div to keep the drop header elsements in a single line. Inside the drop-header div I need to put a heading div and a button element.

Div heading will become my heading. Since regular headings can be buggers with their default settings. To accomplish this I take advantage of the role=heading and that means I will also a aria-level to tell screen readers what heading level this heading is. I will use css the style this myself.

The button element is contained within the drop-head to the right of the heading. Button is the control action item. It also contains and img element. The aria-expanded attribute of the button will determine if initially expanded or collapsed. The aria-controls must be set to the drop panel id. The img element wil point to a file called a name with the word "collaps" or "expand" in it. This way we simply replace the words with each other to swap the images. I will use sg files for this.

Lastly drop-content div which will contain the actual content that the drop down will expose when expanded and hide when collapsed. It must have the aria-hidden attribute. We will toggel the value from true or false to hide it from screen readers as well. We do not need a class to hide it, since the drop panel will resize and hide it from eyesight.

Step 3c Wire up some buttons - click event

Create a function called initializeDrop()" move the alertStatus querySelector code here. Now replace all the code between the curly braces with "document.addEventListener("DOMContentLoaded",(e) => {initializeDrop();})". Now youou will have muliple drop panels with drop-action-buttons on them. They all will have a name attribute. So we need an array of of these buttons so that we can attach something to the click event.

Step 4 Error handling, navigation, & misc
Step 4a Error Handling

You always have the ability to sayAlertStatus(msg to get some kind of feed back. It is so easy. Add sayAlertStatus line save, reload browse. Now you sould hear it. That easy and fast.

Try catch are always great at catching some of those elusive errors. So add them especially in more complex functionns. The catch error you will want to sayAlertStatus the stack trace (error.stack). Then output error.message. This way when you hear it you can go to it. It is under a Heading level 5. So press the 5 key if you are using a screen reader. Cursor down from there and git the line number and description.

JavaScript is always a bit of a pain. Golden rule is write small and check if it is on track. Javascript can send down an agonizing path if ou go to deep. It is a runtime language and that is one of the dropbacks. I prefer TypeScript which is coding time error checking. That means you catch possible errors earlier and it forces you to learn good programming habits. Also as a blind programmer I love the completion capabilities in vsCode that are not as accurate in JavaScript.

Step 4b Navigation

I cannot emphasize how import code navigation is when developing. It would be impossible to do some of the more complex projects I have been coding. The HTML on this project is so large and complicated in navigation, it took all my abilities and then some.

Object navigation is awesome. HTML you can object navigate by ids or classes. You can enter "#drop-r02" or ".classname" or a tagname. So finding your tag is easier. css Object navigation lists all the media selectonrs. So jump right to

Brace matching (ctrl+shift+"\")). CSS and js files are much easier when you use brace navigation. Press ctrl+shift+"\" and it will find matching pairs for any of the following: {}[](). Makes it really easy to help track down those pesky " errors where you are missing are too many closing} ' or other. I simply press this key when on the brace and go to the matching one. If it is not where I was supposed to end up, I need to add or subtract a "}" to balance it.s

The final useful navigation/edit super handy feature is folding code. Sighted use that mouse to fold code, copy, cut, move, paste. We simply fold the code to 2 lines of code. selct both lines and copy, cut, or just move past it. Since it is folded you only have to cursor past 2 lines and not 20 lines of code. Also nice feature you know folded code is balanced at the outer matching items. HTML folding is absolutely neccessary and not that possible without utilizineg it. This page has many lines and sections of html, so I utilized folding alot.

Key Handling

I added a windows key-up event listener in the initializeDrop() function. You use key-up, because their are too many events on key-down and the user maybe in the middle of pressing a modifier key (alt, ctrl, shift, ...). As with almost all event listeners you create, you will want to prevent default proccessing of that event. Otherwise, other things will happen. Sometimes that is desirable, so try it with and without. So in most cases it is e.preventDefault(); .
